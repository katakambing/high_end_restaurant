High-End Restaurant Reservation System

Project Type: Flutter Mobile Application (Android)
Database: Cloud Firestore (Firebase)
Architecture: Service-based with Role-Based Access Control (Guest / User / Admin)

Section A: Requirements Compliance Check

# Requirement Status Implementation Details
1 Guest Page with menu packages ✔ GuestHome — Visually appealing cards with images, descriptions, and prices. Accessible without login.
2 Guide to register/login ✔ "Log In" button in guest banner. Snackbar prompts login when guest taps "Book Now".
3 Separate/unified login with roles ✔ Firebase Auth + Firestore role field. Root widget checks role and routes to User Home, Admin Home, or Guest Home.
4 Menu browsing (User) ✔ HomeScreen with search bar and category ChoiceChips filter (All, Burgers, Snacks, Drinks).
5 Booking form (date, time, guests) ✔ BookingPage — Native date picker, time picker, +/- guest counter with real-time updates.
6 Real-time price: (base × guests) + service charge ✔ BookingPage calculates: baseTotal = price × guests, serviceCharge = baseTotal × 10%, grandTotal = baseTotal + serviceCharge. Displayed in price breakdown card.
7 Summary & Confirmation screen ✔ BookingConfirmationPage — Success icon, full booking summary card (package, date, time, guests, total, status), and "Back to Home" button.
8 View My Reservations ✔ MyReservationsPage — BookingCard widget displays Event Date, Menu Package Name, Number of Guests, Total Price, Status badge.
9 Update reservations (re-pricing, change package) ✔ EditBookingPage — Dropdown to change menu package, date/time pickers, +/- guest counter with live re-pricing including service charge.
10 Delete with confirmation dialog ✔ AlertDialog confirmation before deletion in MyReservationsPage.
11 Admin CRUD menu packages ✔ ManageMenuPage (Read/Delete), AddMenuPage (Create), EditMenuPage (Update with name, description, price, image URL, category).
12 Admin manage user accounts ✔ ManageUsersPage — StreamBuilder lists all users from Firestore with email, role badge, and delete capability with confirmation.
13 Admin manage all customer reservations ✔ AdminBookingsPage — View all bookings. Approve/Reject pending bookings, Complete/Cancel approved bookings. Delete any booking with confirmation.
14 Search / Filter functions ✔ Search bar (real-time filtering by name) + Category ChoiceChips on both GuestHome and HomeScreen.
15 ≥ 3 third-party packages ✔ 7 packages: firebase_core, firebase_auth, cloud_firestore, intl (date formatting), cached_network_image (image caching), google_fonts (typography), flutter_svg (SVG support).
16 At least 1 database product ✔ Cloud Firestore with collections: menu, users, bookings.

eventTimeStringEvent time in "HH:mm" format
numGuestsNumber (int)Number of guests

Section F: Key Features

Security Features

  • Firebase Authentication with email/password
  • Role-Based Access Control (RBAC) — users routed based on Firestore role field
  • Guest users cannot make bookings (must login/register)
  • Admin-only pages protected by role check
  • Forgot Password via Firebase email reset

User Experience Features

  • Cached network images for fast loading
  • Real-time search filtering
  • Category-based filtering with ChoiceChips
  • Native date and time pickers
  • Real-time price calculation with service charge breakdown
  • Color-coded booking status badges (Pending=Orange, Approved=Green, Completed=Blue, Cancelled=Red)
  • Confirmation dialogs for destructive actions
  • Success confirmation screen after booking
